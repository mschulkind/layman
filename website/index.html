Why Layman?

A smarter way to manage your tiling window layouts

Per-Workspace Layouts

MasterStack on workspace 1 for coding, Grid on workspace 2 for monitoring, Autotiling on workspace 3 for browsing. Each workspace works the way you want.

Zero-Latency Commands

Use Sway's nop bindings for instant response. No process spawning, no delays — just immediate layout control.

Hot Reload

Change your configuration and reload without restarting. Tweak your layouts on the fly without losing your window arrangement.

Runtime Control

Switch layouts, rotate windows, toggle stack sides, and more — all from keybindings. Full control without touching the config.

Extensible with Python

Create custom layouts without modifying layman. Just drop a Python file in your config directory and it's automatically loaded.

Minimal Footprint

Single daemon, simple TOML config, zero external dependencies beyond Python. Runs quietly in the background.

Built-in Layouts

Three powerful layouts out of the box, plus custom layouts via Python

MasterStack

A primary master window with a configurable stack. New windows become master, pushing the old master to the stack.

  • Configurable master width (1-99%)
  • Stack layouts: splitv, splith, tabbed, stacking
  • Stack side: left or right
  • Rotate windows clockwise/counter-clockwise
  • Visible stack limit with substack

Get Started in 60 Seconds

Install, configure, and start using layman

Installation & Setup 
# Clone and install
git clone https://github.com/frap129/layman
cd layman && pip install .

# Create config
mkdir -p ~/.config/layman
cat > ~/.config/layman/config.toml << 'EOF'
[layman]
defaultLayout = "MasterStack"
masterWidth = 50
EOF

# Add to your Sway config (~/.config/sway/config)
# exec layman
# bindsym $mod+m nop layman layout set MasterStack

# Reload Sway and you're done!
